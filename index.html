3. 投票功能测试

投票功能即将投入使用，请重点测试以下流程。

3.1 管理员/理事发起投票

1登录管理员账号

邮箱：admin@xjtu.sg，密码：admin123

2进入管理后台 → 投票管理

点击顶部导航"管理后台"，然后点击"投票管理"标签页

3点击"发起新投票"

填写以下信息：

  • 投票标题：例如"2026年校友会年度聚餐地点选择"
  • 说明：描述投票背景
  • 投票选项：至少2个选项，可点击"+ 添加选项"增加更多
  • 开始时间：选择开始日期和时间（精确到分钟）
  • 截止时间：选择截止日期和时间（精确到分钟）
  • 参与范围：
    • 点击"全体校友"按钮 → 自动勾选所有成员
    • 点击"仅理事"按钮 → 只勾选管理员和理事
    • 点击"清空"按钮 → 取消所有勾选
    • 可逐人手动勾选/取消，实现精准控制
4确认并发起

点击"发起投票"按钮，投票将立即生效并显示在投票管理列表中

验证点：发起后检查投票列表是否显示新投票，状态为"进行中"，参与范围显示"全体校友"/"仅理事"/"指定成员(N人)"。

3.2 校友参与投票

1退出管理员，登录校友账号

例如：chensiyuan@gmail.com / user123

2进入"投票决策"页面

点击顶部导航"投票决策"，查看"进行中"标签页

3选择选项

点击想投的选项（会高亮显示），此时不会立即提交

4点击"确认投票"按钮

确认后才会真正提交投票。系统提示"投票成功！"

5查看投票结果

投票后显示各选项的票数和百分比，您的选择旁标记"✓ 我的选择"

验证点：投票后检查票数是否正确增加，"已投票"标签是否显示。

3.3 修改投票（截止前）

1在已投票的投票卡片中，点击"修改投票"按钮

进入修改模式，选项恢复可选状态

2选择新的选项，点击"确认修改"

系统提示"投票已修改！"，原选项票数-1，新选项票数+1

3也可点击"取消修改"放弃
验证点：修改前后总票数不变，只是选项间票数转移。

3.4 投票到期自动关闭

1发起一个截止时间设为几分钟后的投票
2等待截止时间到达后刷新页面
3检查投票状态

应显示"已结束"标签，截止时间显示红色"已于 xxxx 截止"，不再显示"修改投票"按钮

验证点：到期后尝试通过API直接投票（应返回"投票已截止"错误）。

3.5 管理员手动结束投票

1登录管理员，进入管理后台 → 投票管理
2点击某个进行中投票旁的"结束投票"按钮
3确认投票状态变为"已结束"

3.6 权限测试

测试场景预期结果
仅理事投票 → 校友账号查看显示"此投票仅限理事参与"，无法投票
指定成员投票 → 未被选中的校友查看显示"此投票仅限理事参与"，无法投票
校友尝试发起投票无"管理后台"入口，无法发起
到期后尝试投票服务器拒绝，返回"投票已截止"

4. 注册与审批流程

4.1 新校友注册

1访问首页，点击"注册"或"立即加入校友会"
2第一步：填写基本信息

姓名*、邮箱*、手机号码*、密码*（≥6位）均为必填

3第二步：填写学业信息

学号、毕业年份、学院、专业

4第三步：职业信息 + 上传证明材料

可上传学位证书、在校证明等（最多5个文件，每个≤10MB，支持PDF/JPG/PNG）

5提交后显示成功页面

提示"理事会将在7个工作日内审核"

验证点：注册后用该邮箱登录 → 应提示"账号审核中"。

4.2 管理员审批

1登录管理员，进入管理后台 → 待审核
2查看申请人完整信息（邮箱/手机/学号/学院/专业/单位/职位/简介）
3查看上传的证明材料（图片直接预览，文件可点击下载）
4点击"通过"或"拒绝"
验证点：通过后用该账号登录 → 应可正常访问所有功能。

5. 管理后台功能

5.1 校友管理

  • 查看所有校友列表（支持搜索姓名/邮箱、按角色筛选）
  • 编辑用户信息（姓名/邮箱/角色/状态）
  • 删除用户（管理员自己不能删除自己）

5.2 新闻管理

  • 发布新闻（标题/分类/内容）
  • 编辑已有新闻
  • 删除新闻

5.3 投票管理

  • 发起新投票（含参与范围精确选人）
  • 查看投票进度和结果
  • 手动结束投票
  • 删除投票

6. 新闻动态

1登录任意已通过账号，点击"新闻动态"
2浏览新闻列表，支持按分类筛选（母校动态/活动回顾/重要通知/校友故事/活动通知）
3点击新闻卡片查看详情

7. 互动论坛

1点击"互动平台"进入论坛
2发布新帖 → 填写标题、选择分类、输入内容

可上传附件（图片/文件，最多5个，每个≤10MB）

3查看帖子详情，发表回复
4删除自己的帖子（管理员可删除任何帖子）
验证点：帖子中的图片附件应在详情页显示缩略图预览。

8. 会费缴纳

8.1 校友缴纳会费

1登录校友账号，点击导航栏"会费缴纳"
2查看会员资格状态

页面顶部显示会员起始日期和到期日期（如：2026年1月1日 至 2026年12月31日），以及当前状态（有效/已过期）

3查看当年度会费信息

显示年度、金额（如 SGD 50）、截止日期，以及PayNow转账信息（号码、收款人、金额、二维码）

4通过银行App扫码/手动转账后，上传付款截图

点击上传区域或拖拽截图文件，填写备注（选填），点击"提交缴费凭证"

5提交后状态变为"已提交，等待确认"（黄色）
验证点：提交后刷新页面，确认状态显示为"待确认"；页面底部"缴费记录"表格中有新记录。

8.2 管理员确认缴费

1登录管理员账号，进入"管理后台 → 会费管理"
2查看缴费记录列表，点击凭证链接查看校友上传的付款截图
3核实无误后点击"确认"按钮

确认后系统自动将该校友的会员资格延长一年

验证点：确认后切换到校友账号，查看会费页面 → 状态应变为绿色"已缴纳"，会员到期日期自动延长一年。

8.3 管理员设置会费标准

1在"管理后台 → 会费管理"顶部的"会费设置"区域

可修改：年度、金额、货币、PayNow号码、收款人名称、说明、截止日期

2点击"保存设置"，所有校友看到的会费信息将即时更新

9. 个人中心

1点击右上角头像 → "个人中心"
2编辑个人资料：姓名、邮箱、手机、单位、职位、简介
3修改密码：输入旧密码 + 新密码
4查看"我的活动"：历史投票参与记录、论坛发帖记录

10. 校友名录

1点击"校友名录"
2搜索校友（按姓名/学院/专业）
3按毕业年份或学院筛选
4点击校友卡片查看详细信息

11. 多用户并发测试

本系统使用SQLite服务端数据库，支持多用户同时使用。

测试方法

1打开两个不同的浏览器（或一个正常窗口 + 一个隐身窗口）
2分别登录不同账号（如管理员和校友）
3在一个窗口发起投票，在另一个窗口刷新查看是否出现
4两个账号同时对同一投票投票，检查票数是否正确
预期结果：数据实时同步，票数准确，不会出现重复投票。

12. 发现Bug？通过"互动平台"提交

如果您在测试过程中发现任何问题或Bug，请通过系统内置的"互动平台"（论坛）提交反馈：

1登录您的测试账号
2点击顶部导航"互动平台"
3点击"发布新帖"
4填写Bug报告
  • 标题：简要描述Bug，例如"【Bug】投票页面在手机上显示异常"
  • 分类：建议选择"综合讨论"或其他合适分类
  • 内容：请包含以下信息：
    • 您使用的账号角色（管理员/理事/校友）
    • 操作步骤（一步一步描述您做了什么）
    • 期望结果（您认为应该发生什么）
    • 实际结果（实际发生了什么）
    • 使用的浏览器和设备
  • 附件：可上传截图或屏幕录屏，帮助开发者快速定位问题
5点击"发布"提交
提示：管理员会定期查看论坛中的反馈帖子，并在帖子下方回复处理进展。请关注您发布的帖子的回复通知。
